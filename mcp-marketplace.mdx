## Overview

SINT Protocol integrates with the **Model Context Protocol (MCP)** — the open standard for connecting AI models to external tools and data sources. SINT adds a security and governance layer on top of MCP, ensuring every tool invocation is authenticated, authorized, constrained, and logged.

<Info>
**Packages:** `@sint/bridge-mcp` (bridge adapter), `sint-mcp` (MCP server app), `sint-mcp-scanner` (discovery tool)
</Info>

---

## How It Works

Standard MCP exposes tools to AI agents without governance. SINT wraps MCP with the Policy Gateway:

```
AI Agent → MCP Client → SINT MCP Server → Policy Gateway → MCP Tool Server
                              ↕                                    ↕
                        Evidence Ledger                       Tool Result
```

Every tool call passes through the intercept pipeline — capability token verification, constraint checking, tier enforcement, and evidence logging — before reaching the target tool server.

## SINT MCP Server

The `sint-mcp` app (`apps/sint-mcp/`) exposes SINT Protocol as an MCP-compatible server. Any MCP client (Claude, OpenAI, Cursor, etc.) can connect to it and access SINT-governed resources.

### Capabilities Exposed

| MCP Resource | SINT Feature | Description |
|:-------------|:-------------|:------------|
| `sint/intercept` | Policy Gateway | Submit an action for policy evaluation |
| `sint/tokens` | Capability Tokens | Create, delegate, revoke tokens |
| `sint/ledger` | Evidence Ledger | Query audit trail, generate proofs |
| `sint/health` | Gateway Health | System status and readiness |
| `sint/schemas` | Schema Catalog | Available validation schemas |

### Starting the MCP Server

```bash
cd apps/sint-mcp
pnpm build
pnpm start
```

Or use with Claude Desktop / any MCP client:

```json
{
  "mcpServers": {
    "sint": {
      "command": "node",
      "args": ["apps/sint-mcp/dist/index.js"],
      "env": {
        "SINT_GATEWAY_URL": "http://localhost:4100"
      }
    }
  }
}
```

## MCP Scanner

The `sint-mcp-scanner` app discovers and catalogs MCP servers across your environment:

```bash
cd apps/sint-mcp-scanner
pnpm build
pnpm start
```

It scans for MCP servers and registers discovered tools with the SINT gateway for governance tracking.

## Capability-Scoped Tool Discovery

Unlike standard MCP where all tools are visible to all agents, SINT filters tool discovery based on the agent's capability token:

<Steps>
  <Step title="Agent connects to SINT MCP Server">
    The agent presents its Ed25519 identity and capability token.
  </Step>
  <Step title="Token is validated">
    The Policy Gateway verifies the signature, checks expiry, and evaluates delegation chain.
  </Step>
  <Step title="Tools are filtered">
    Only tools matching the token's `resource` and `action` scope are returned in the tool list.
  </Step>
  <Step title="Invocations are governed">
    Each tool call passes through the full intercept pipeline with constraint enforcement.
  </Step>
</Steps>

This means an agent with a token scoped to `resource: "robot-arm-1"` will only see tools relevant to that robot arm — not the entire tool catalog.

## Security Guarantees

Every MCP tool invocation through SINT inherits the full security stack:

| Layer | Protection |
|:------|:-----------|
| **Authentication** | Ed25519 agent identity verification |
| **Authorization** | Capability token scope matching |
| **Constraints** | Physical limits enforced per-invocation |
| **Audit** | Every tool call logged to evidence ledger |
| **Anomaly detection** | Goal hijack scanner on tool arguments |

<Warning>
MCP tool arguments are passed through the goal hijack detector. If tool arguments contain patterns matching prompt injection (25+ regex patterns, 0.6 confidence threshold), the request is denied and flagged.
</Warning>
