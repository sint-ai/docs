## Overview

**SINT Bridge** is the integration layer that connects AI agents to external systems — APIs, blockchains, robots, and industrial protocols — through a single governed interface. Instead of building custom integrations for each target system, agents interact with SINT Bridge via the [Model Context Protocol (MCP)](/bridges) and the bridge handles authentication, routing, policy enforcement, and audit logging.

The core principle: **one connector spec, any target system**. Swap a URI, not your code.

---

## How It Works

```
Agent → JSON-RPC Request → SINT Bridge → [Auth + Policy Check] → Target System
                                ↕                                      ↕
                          Evidence Ledger                         Response
```

<Steps>
  <Step title="Agent sends a request">
    The AI agent sends a JSON-RPC request describing the action — e.g., `sendAsset`, `queryDatabase`, `controlRobot`.
  </Step>
  <Step title="Bridge routes and authenticates">
    SINT Bridge selects the appropriate connector (REST, GraphQL, gRPC, ROS2, MAVLink, blockchain RPC) and handles authentication (OAuth 2.1, API keys, wallet signing).
  </Step>
  <Step title="Policy enforcement">
    Every request passes through the [Policy Gateway](/gate) — capability token verification, constraint checking, and tier enforcement.
  </Step>
  <Step title="Evidence logging">
    The decision (approve/deny/escalate) is appended to the [Evidence Ledger](/security#evidence-ledger) with full context.
  </Step>
  <Step title="Response returned">
    The target system's response is streamed back to the agent with updated context.
  </Step>
</Steps>

## Supported Connectors

### Protocol Bridges (Built-in)

| Bridge | Package | Target Systems |
|:-------|:--------|:---------------|
| **ROS2** | `@sint/bridge-ros2` | Robot arms, mobile robots, drones (via ROS2) |
| **gRPC** | `@sint/bridge-grpc` | Microservices, ML model servers |
| **MCP** | `@sint/bridge-mcp` | Any MCP-compliant tool server |
| **A2A** | `@sint/bridge-a2a` | Google Agent-to-Agent protocol |
| **MQTT/Sparkplug** | `@sint/bridge-mqtt-sparkplug` | Industrial SCADA, IoT hubs |
| **OPC-UA** | `@sint/bridge-opcua` | PLCs, SCADA, manufacturing systems |
| **MAVLink** | `@sint/bridge-mavlink` | Drones, UAVs, ground vehicles |
| **OpenRMF** | `@sint/bridge-open-rmf` | Multi-robot fleet management |
| **IoT** | `@sint/bridge-iot` | Generic IoT devices |
| **Economy** | `@sint/bridge-economy` | Metered resource tracking |
| **Swarm** | `@sint/bridge-swarm` | Multi-agent coordination |

### Web2 Integration

SINT Bridge handles Web2 authentication through standard OAuth 2.1 flows:

- **Enterprise SaaS** — Salesforce, HubSpot, Google Workspace, Slack, Notion
- **Social platforms** — LinkedIn, X/Twitter, Instagram
- **Developer tools** — GitHub, Jira, Linear
- **Communication** — Email (SMTP/IMAP), SMS, voice

### Web3 Integration

Chain-agnostic blockchain connectivity:

- **EVM chains** — Ethereum, Polygon, Arbitrum, Base
- **Solana** — SPL tokens, DeFi protocols
- **Cosmos** — IBC cross-chain messaging
- **Other** — Polkadot (XCM), LayerZero cross-chain

## Security Model

Every bridge interaction inherits the full SINT security stack:

| Layer | Protection |
|:------|:-----------|
| **Authentication** | Ed25519 agent identity + OAuth 2.1 for external services |
| **Authorization** | Capability tokens with attenuation-only delegation |
| **Constraints** | Physical limits (force, velocity, geofence) enforced per-request |
| **Audit** | SHA-256 hash chain evidence ledger for every action |
| **Governance** | Five-tier safety system with human-in-the-loop escalation |

<Warning>
All bridge connectors enforce policy at the request level. There is no "passthrough" mode. Even read-only operations are logged to the evidence ledger.
</Warning>

## Agent-to-Agent (A2A) Protocol

SINT Bridge includes native support for Google's Agent-to-Agent protocol, enabling secure inter-agent communication:

- **Agent card discovery** — Register and discover agents via `/v1/a2a/agents`
- **Message routing** — Send messages between agents with capability token delegation
- **Task coordination** — Delegate and track multi-agent workflows

Think of MCP as the **agent ↔ tools** interface and A2A as the **agent ↔ agent** interface. SINT Bridge bundles both under the same governance model.
